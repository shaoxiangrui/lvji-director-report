# 驴迹产品总监汇报页 HTML Design

日期：2026-03-08

## 目标

将 [驴迹AI赋能报告-精筛版-务实简洁版](/Users/wenjiayan/PycharmProjects/驴迹AI/docs/驴迹AI赋能报告-精筛版-务实简洁版.md) 转化为一个适合面向产品总监汇报的独立 HTML 页面。

页面目标不是展示全部材料，而是支持总监在 8-10 分钟内完成三项判断：

1. 战略方向是否成立。
2. 第一阶段优先做什么。
3. 该方案是否值得投入。

## 设计结论

采用 `Boardroom Atlas` 方向。

这是一个把“董事会简报”与“文旅地图叙事”融合的页面风格。页面既要有高层汇报的稳重感，也要保留驴迹所在行业的路径、地图、场景感，避免做成通用 AI 官网或运营后台。

## 视觉风格

### 关键词

- `Boardroom Atlas`
- `Editorial Cartography`
- `Strategic Brief`

### 视觉语言

1. 使用温暖纸面色作为主背景，不使用纯白或深色科技底。
2. 使用深墨绿作为主文字和结构主色，传达经营、判断和行业沉稳感。
3. 使用铜金 / 琥珀作为强调色，用于优先级、决策点、收益提示。
4. 使用低透明度地图网格、路径线和等高线纹理建立文旅识别度。
5. 动效克制，只服务“阅读节奏”和“章节定位”。

### 字体策略

1. 标题使用中文衬线风格字体，增强正式汇报感。
2. 正文使用人文无衬线字体，保证长内容阅读效率。
3. 数字、优先级标签和关键指标使用等宽数字字体。

## 信息架构

### 页面顺序

1. `Hero / Executive Overview`
2. `P0 / 战略定位`
3. `P0 / 关键场景`
4. `P1 / 用户故事与执行`
5. `P2 / 调研与投资依据`
6. `Decision Board`

### 排序原则

1. 先结论，后证据。
2. 先战略，后功能。
3. 先 `P0`，再 `P1/P2`。
4. 每一屏只回答一个管理问题。

## 模块清单

### M1. Atlas Nav

固定导航，显示章节、优先级和当前阅读位置。

### M2. Executive Hero

承载战略判断、产品定义、4 个核心结论、4 个待拍板问题，以及一条地图式战略路径线。

### M3. Strategy Spine

承载客户问题、统一问题陈述、定位陈述，以及 `1 个中枢 + 4 类数字员工` 的主结构。

### M4. Boundary Ledger

以双栏方式展示“可以做到”和“不能承诺”，明确边界。

### M5. Roadmap Rail

以 `P0 / P1 / P2` 三段轨道形式展示 12 个月路线图。

### M6. Scenario Trio

三张关键场景卡，统一采用：

1. 变化前
2. AI 介入
3. 变化后
4. 企业价值

### M7. Value Roles Strip

展示 3 类关键角色的关注点和 AI 价值。

### M8. Journey & Validation Board

展示 6 节点旅程和问题验证到商业验证的闭环。

### M9. Epic Board

展示 4 个 Epic 及成功指标。

### M10. Story Map Matrix

用横向矩阵展示 Backbone 与 `Release 1 / Release 2 / Release 3`。

### M11. Priority Ladder

独立展示 `P0 / P1 / P2` 的优先级和目标。

### M12. Evidence Grid

展示公司关键信号、AI readiness 和第一优先行动项。

### M13. Investment Board

展示 TAM / SAM / SOM 和“做 / 不做”投资判断。

### M14. Final Decision Board

页面最后停在总监待拍板问题、建议动作和下一阶段里程碑。

## 交互规范

### 允许的交互

1. 粘性导航。
2. Scroll Spy 高亮当前章节。
3. 首屏战略路径线渐进显现。
4. 卡片轻度浮现和进入动画。
5. 路线图阶段依次点亮。

### 不做的交互

1. 大面积视差。
2. 粒子、发光、3D 翻转。
3. 高频数字跳动。
4. 复杂切屏。

## 信息展示规范

1. 每屏只设置一个主标题。
2. 每张卡尽量控制在 `3-5` 行核心信息。
3. 优先展示结论、价值、优先级、边界和待决策项。
4. 方法过程、理论框架和背景说明降级展示。
5. `P0` 视觉占比应超过整页的 60%。

## 技术约束

1. 输出为独立 HTML 文件，不覆盖现有 [index.html](/Users/wenjiayan/PycharmProjects/驴迹AI/index.html)。
2. 使用单文件静态 HTML，包含内联 CSS 和少量原生 JavaScript。
3. 页面应支持离线打开，不依赖构建流程。
4. 优先兼容 `1440px` 汇报屏，同时兼顾手机阅读。
5. 内容直接来自母稿，不引入远程数据。
