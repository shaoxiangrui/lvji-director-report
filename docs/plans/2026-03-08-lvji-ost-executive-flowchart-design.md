# 驴迹总监汇报页 OST 流程图化 Design

日期：2026-03-08

## 目标

重构 [lvji-director-report.html](/Users/wenjiayan/PycharmProjects/驴迹AI/lvji-director-report.html) 中 `Opportunity Solution Tree` 的视觉样式。

本次只调整展示方式，不改变内容、顺序和叙事定位。

## 设计判断

当前 `OST` 已经具备“目标 -> 机会 -> 方案 -> 实验”的信息结构，但视觉上仍然更像四列表的战略板，而不是流程图。

用户希望参考流程图布局，但继续保留当前页面的：

1. 米白纸面底色
2. 墨绿主色
3. 铜色强调
4. 董事会简报感

因此，这次不采用 Mermaid 默认风格，也不转成白板图，而是改成 `Executive Flowchart`。

## 视觉方向

### 名称

`Executive Flowchart`

### 概念

像高层经营会上使用的流程图：

1. 左侧一个总目标节点
2. 中间分布机会节点
3. 再向右连接方案节点
4. 最右连接实验节点

整体更像“战略动作流图”，而不是“信息卡片分栏”。

### 风格约束

1. 保留当前页面的纸面质感和地图感纹理。
2. 节点边框更清晰，形状更接近流程图矩形。
3. 圆角比当前卡片略收紧，减少“卡片感”，增强“节点感”。
4. 连接线和箭头更明确，但只做轻线条，不做浓重工程图样式。

## 布局方式

### 桌面端

采用“左侧目标 + 右侧四条主路径”的流图布局。

结构：

1. 左侧目标节点纵向居中。
2. 右侧按路径排列：
   - 机会1 -> 方案1 -> 实验1
   - 机会2 -> 方案2 -> 实验2
   - 机会3 -> 方案3 / 方案4 -> 实验3 / 实验4
   - 机会4 -> 方案5 -> 实验5
3. 目标与机会之间通过一条主干线连接。
4. 方案与实验之间采用标准箭头连接。

### 平板端

改为两段式：

1. 目标单独成块
2. 每条路径按行排列

### 手机端

改为单列流程堆叠：

1. 目标
2. 路径 1-4 依次展示

## 组件设计

### 1. Flowchart Shell

整个 `OST` 区块的外层容器，保留页面一致的纸面背景和边框。

### 2. Target Node

左侧目标节点，视觉最重。

特点：

1. 节点更大
2. 墨绿描边
3. 背景保持浅米白

### 3. Flow Nodes

机会、方案、实验节点统一为流程图框。

规则：

1. 机会节点略偏基础色
2. 方案节点保持中性纸面色
3. 实验节点用铜色轻强调

### 4. Connectors

使用 SVG 或 CSS 连接线实现更接近流程图的箭头关系。

原则：

1. 桌面端箭头明确可见
2. 窄屏端允许退化为布局上的顺序关系
3. 不引入复杂动画

## 预期效果

改完后，这块会更接近用户给的参考图，但仍然是同一套汇报页面的一部分：

1. 结构一眼看出是流程关系
2. 内容仍然是董事会简报语言
3. 不会突然跳成另一套配色或风格
